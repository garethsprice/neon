Neon Cloud

Git-like versioning and collaboration for music applications

Features

Track Management

Create, load, and manage music projects with full metadata support including name, description, and thumbnails.

Commit History

Git-like versioning with commit messages, parent tracking, and the ability to browse and load historical versions.

Smart Diffing

Configurable state diffing detects changes to patterns, parameters, arrangements, and more.

Social Features

Built-in likes, play counts, and remix tracking with real-time notifications.

Real-time Sync

Integrates with WebsimSocket for live updates across all connected users.

Framework Agnostic

No UI framework required. Provides data and events for you to render however you like.

Quick Start

import { CloudStore } from './neon-cloud/index.js';

// Create a store
const store = new CloudStore({
    room: new WebsimSocket(),
    getCurrentUser: () => websim.getCurrentUser()
});

// Listen for events
store.on('committed', ({ track, commit }) => {
    console.log(`Saved: ${commit.message}`);
});

store.on('loaded', ({ data }) => {
    myApp.deserialize(data);
});

// Save your project
await store.commit(
    myApp.serialize(),
    { name: 'My Track', stats: { bpm: 120 } }
);

// Load a track
await store.loadTrack(trackId);

API Reference

CloudStore

new CloudStore(options)
Create a new store. Options: room (WebsimSocket), getCurrentUser (function), generateCommitMessage (async function), diffConfig (object), collectionPrefix (string)
store.commit(data, metadata, options)
Save the current state. Returns { track, commit, message, isNewTrack, isRemix } or { error, message }
store.loadTrack(trackId)
Load the latest version of a track. Returns { track, commit, data }
store.loadCommit(commitId)
Load a specific commit (for history browsing). Returns { track, commit, data, isHistory }
store.checkForChanges(currentData)
Check if current data differs from last commit. Returns { changes, hasChanges }
store.toggleLike(trackId)
Like or unlike a track. Returns { liked: boolean }
store.recordPlay(trackId)
Record a play event for analytics
store.deleteTrack(trackId)
Delete a track and all its commits (owner only)
store.setVisibility(trackId, 'public' | 'private')
Change track visibility (owner only)
store.getFeed({ filter, page, pageSize, sortBy })
Get paginated track list with stats. Returns { items, page, total, totalPages }
store.getTrackHistory(trackId)
Get all commits for a track (newest first)
store.getTrackStats(trackId)
Get stats: { likes, plays, commits, remixes }

Events

Subscribe with store.on(event, callback):

change committed loaded liked unliked played deleted visibilityChanged message reset

Integration Example

// Example: Synth app using neon-cloud
import { CloudStore } from './neon-cloud/index.js';

const store = new CloudStore({
    room: new WebsimSocket(),
    getCurrentUser: () => websim.getCurrentUser(),
    diffConfig: {
        scalarFields: ['name', 'bpm', 'key', 'scale'],
        objectFields: ['patches', 'sequences', 'mixerSettings'],
        arrayFields: ['songStructure']
    },
    // Custom AI commit messages
    generateCommitMessage: async (changes) => {
        if (changes.isInitial) return 'Initial patch';
        return changes.summary[0] || 'Updated';
    }
});

// React to loaded data
store.on('loaded', ({ data }) => {
    synth.loadPatch(data.patches);
    sequencer.load(data.sequences);
    mixer.setLevels(data.mixerSettings);
    ui.refresh();
});

// Auto-save indicator
store.on('change', () => {
    const { hasChanges } = store.checkForChanges(app.serialize());
    saveButton.disabled = !hasChanges;
});

Data Collections

The library uses these WebsimSocket collections (prefixed with collectionPrefix if set):

  • track_v1 - Track metadata (name, owner, head_commit_id, stats, etc.)
  • commit_v1 - Commits (track_id, parent_id, author, message, data)
  • like_v1 - Likes (track_id, username)
  • play_v1 - Play events (track_id)
